Bitcoin Sentiment Analysis

AI/ML NLP Python

Enough talking about bitcoin price, what about the sentiment?

With the rise and fall of bitcoin price, one could argue that the public sentiment also play a great role in it's price. So we incorporate public sentiment as one of the variable to predict bitcoin price by using post from Reddit and X (formerly Twitter) as our dataset with Transformer Model as our architecture of choice!

Data Labelling

Data labelling is done by manually labelling text between Negative (-1), Neutral (0), and Positive (1). It's done for the purpose of training the model to decide which classification it should conclude. An already labelled dataset from SurgeAI is combined too resulting in 3.803 data. With distribution of 748 Negative, 1.177 Neutral, and 1.818 Positive.

Data Preprocessing

One of the important part of creating a text classification model is how does one process the data before hand, so 6 step of data preprocessing is used. Which are

  • Cleansing of urls and HTML Entity
  • Lowercasing
  • Tokenization
  • Cleaning of Punctuation and Conjunctions
  • Cleaning of Special Character and Number
  • Lemmanization

After all that's set and done, we delete tweets that's categorized as spam. It's done by manually detecting tweets that's similar and deleting tweets that similiar to it. For example tweet like

Data Augmentation

Data Augmentation Count using Back-Translation

Before going into Data Augmentation, undersampling is done by taking the data that have the lowest count so the model doesn't skew into particular classification. Hence our original data come down from 3.803 to 1.536 with 512 data across all classification.

Data Augmentation are done by using back-translation, which are by translating the text into another language and then translating it back to english. It's done to enrich and multiply the amount of data we have to train the model. We use google translate and a Natural Language Processing model from University of Helsinki.

Conclusion

Well it ain't much, but it's honest work! An empirical study was conducted using four text classification model and dataset with various enriching method and tool, which are Back-Translation using Top 10 & 20 language with the highest BLEU score on Google Translate, Back-Translation using HelsinkiNLP, and EDA_NLP by Jason Wei. It is found that using Back-Translation using Top 10 language with the highest BLEU score on Google Translate created the best model over four that's experimented on, but it doesn't mean it's always the best over every case.
